## Findings

### 1. HIGH — `_render_frame` heap overflow on long-cluster input *(fixed)*

**File**: `src/animate.cyr`, `_render_frame` cluster-bytes copy loop (pre-fix lines 263–272).

**Issue**: `_render_frame` writes a full cluster's bytes (`clen` = `next_off - off`) into `line_buf` (32 768 B), then checks `pos + ANUENUE_FLUSH_RESERVE > ANUENUE_LINE_BUF` *after* the write. A grapheme cluster's byte length is unbounded in anuenue's practical-subset classifier — a base codepoint followed by a long run of combining marks, variation selectors, ZWJ chains, or paired regional indicators is **one cluster** at pre-tag time.

`ANUENUE_ANIMATE_INPUT_MAX` is **65 536 bytes** — twice `ANUENUE_LINE_BUF`. So an input shaped as `[base char][N × U+0301]` with `N ≈ 32 500` produces:

- pre-tag: 1 cluster, `ctab[0] = 0`, sentinel `ctab[1] = 65 001`
- `_render_frame` i=0: `clen = 65 001`
- write: `_emit_phase_esc` (~19 bytes for TRUECOLOR) + 65 001 cluster bytes → `pos = 65 020`
- the buffer is 32 768 bytes; **`pos = 65 020` is a 32 252-byte heap overflow past `line_buf`'s allocation**

Heap layout in `anuenue_animate` puts `_PHASE_ESC_TABLE` (~48 960 B) immediately after `line_buf`, so overflow corrupts the phase-escape table and the subsequent `_emit_phase_esc` calls would emit garbage escape bytes from corrupted table entries. With a deeper overflow, the bump allocator's bookkeeping could be hit.

**Reproduction** (pre-fix):

```sh
{ printf 'A'; for i in $(seq 1 32500); do printf '\xcc\x81'; done; } | anuenue -a -d 1
```

Adversary control: the input bytes only — pipe-purity means there's no auth or path requirement. The filter loop (`anuenue_filter`) is **not** affected — it writes one codepoint per iteration (`seqlen ≤ 4`) and checks `FLUSH_RESERVE` between each, so its worst-case write per iteration is 23 bytes.

**Fix**: mid-cluster flush in `_render_frame`. Inside the cluster-bytes copy loop, check `pos + ANUENUE_FLUSH_RESERVE > ANUENUE_LINE_BUF` *per byte* (not just after the cluster). When the check fires mid-cluster:

1. Reset SGR (`tty_sgr_reset_buf`) so the partial output is colour-balanced.
2. Flush `line_buf` to stdout.
3. Re-emit the same `phase` escape via `_emit_phase_esc` so the next bytes render under the same colour.

The visible behaviour is identical to a short cluster — the user sees one cluster rendered in one colour. Internally we just split the write across multiple `file_write` calls when the cluster is bigger than `LINE_BUF - FLUSH_RESERVE - escape_size`.

**Test added**: `tests/anuenue.tcyr` group `M8 audit — _render_frame long-cluster bound`. Builds an 8 KB cluster (base + many combiners) in-memory, renders it through `_render_frame` with a stub stdout-capture, asserts:

- final `pos` is bounded by `LINE_BUF`
- the cluster bytes are all present in the captured output
- the number of mid-cluster flushes is correct (`ceil(clen / (LINE_BUF - FLUSH_RESERVE - escape_size))`)

Plus a golden-equivalent assertion: `cluster_bytes_in == cluster_bytes_out` (every input byte makes it to stdout, no truncation).

**Acceptance**: the post-fix code passes the new test; existing 6 golden fixtures byte-identical (small-cluster paths unchanged); animate-smoke green.

### 3. INFO — Filter loop bounds (`anuenue_filter`)

The hot-path filter loop in `src/filter.cyr` writes one codepoint per iteration:

- ASCII short-circuit: ≤ 20 bytes/iter (`_emit_phase_esc` worst-case 19 + 1 char byte).
- Multi-byte: ≤ 23 bytes/iter (`_emit_phase_esc` 19 + `seqlen ≤ 4`).
- LF branch: writes `tty_sgr_reset_buf` (4 B) + 1 LF byte, then flushes immediately.
- EOF carry-len fallback: ≤ 3 iterations of ≤ 20 bytes each.

`ANUENUE_FLUSH_RESERVE = 32` covers the worst-case 27-byte append (23 + 4-byte reset). The check fires *after* every codepoint write, so `pos ≤ 32 736` is the post-iter invariant before the next codepoint can be appended. Adversarial UTF-8 doesn't change this because `utf8_seq_len` is capped at 4.

`_phase_esc_init` payload is bounded by darshana's `tty_fg_rgb_buf` / `tty_fg_256_buf` / `tty_sgr_buf` return values; all three produce escapes ≤ 24 bytes, within the 24-byte payload region of the 32-byte entry slot.

**No action required.** This is the path most likely to be hit in real production (MOTD pipelines) and it is well-bounded.

### 4. INFO — UTF-8 surface

`utf8_seq_len` (`src/filter.cyr:260`) returns ∈ {0, 1, 2, 3, 4}; never larger, never negative.

- Invalid leading bytes (0x80–0xC1, 0xF5–0xFF) return 1 → byte emitted as 1-byte cluster.
- Truncated at chunk boundary (need N bytes, have < N) → returns 0; filter loop carries the partial bytes to the next read (`carry_len`, max 3).
- Invalid continuation (`(byte & 0xC0) != 0x80`) → returns 1 → graceful per-byte degradation.

`utf8_decode` (`src/filter.cyr:295`) is called only with `seqlen ∈ {1..4}` after `utf8_seq_len` has validated. No OOB read.

Surrogates (U+D800..DFFF) and codepoints above U+10FFFF can be produced from a malformed 3-byte or 4-byte sequence respectively; both are tolerated by `cp_is_extending` (binary search returns 0 for out-of-range codepoints) and `cp_is_regional_indicator` (range check). Output bytes echo the input bytes verbatim, satisfying the "never panic, never over-read" rule.

**No action required.** Behaviour matches [ADR 0003 § Consequences](../adr/0003-grapheme-cluster-cycling.md).

### 5. INFO — Cluster table sentinel

`_pretag_clusters` (`src/animate.cyr:110`) writes `n_clusters` entries into `ctab` plus one sentinel at `ctab + n_clusters * 8`. `ctab` is allocated as `(ANUENUE_ANIMATE_CLUSTER_MAX + 1) * 8 = 65 544 bytes`, so even at `n_clusters = max_clusters` the sentinel slot fits.

The loop guard `i < n_bytes && n_clusters < max_clusters` prevents `n_clusters` from running past `max_clusters`. Excess input bytes (beyond what the cluster cap can index) are silently dropped from the animation render — graceful degradation, no OOB.

**No action required.**

### 6. INFO — Phase arithmetic overflow

`ANUENUE_PHASE_START = flags_get_int(fs, f_seed) + flags_get_int(fs, f_offset)` adds two user-supplied i64 values. Worst-case overflow (`i64::MAX + 1`) wraps to a negative value. Cyrius integer arithmetic is two's-complement wrap on overflow; no UB.

`_emit_phase_esc` normalizes `phase` via `phase % ANUENUE_PHASE_MOD; if (p < 0) { p = p + ANUENUE_PHASE_MOD; }` before indexing `_PHASE_ESC_TABLE`. The modulo operation is well-defined on the full i64 range; the result `p` is in `[0, ANUENUE_PHASE_MOD)` after the correction, so the table index is always in-bounds.

`hsv_rainbow` performs the same normalization. The hot-loop `phase = phase + ANUENUE_PHASE_STEP` accumulates but is normalized at every `_emit_phase_esc` call.

**No action required.** The rainbow may be visually surprising at extreme seeds but never OOB.

### 7. LOW — `-d` value with no upper bound

`anuenue -a -d <int>` accepts any positive i64. The deadline arithmetic is:

```
deadline_ns = start_ns + duration_secs * 1000000000;
```

`duration_secs * 1e9` overflows when `duration_secs > i64::MAX / 1e9 ≈ 9.2e9` (about 292 years). Beyond that, the multiplication wraps to a negative value; `deadline_ns = start_ns + (negative)` lands below `start_ns`. The frame loop then sees `clock_now_ns() >= deadline_ns` immediately and exits cleanly.

This is graceful degradation — pathological input doesn't crash, just terminates the animation early. Not exploitable.

**Action**: document the bound (~ 292 years) in `--help` / animation example notes if a user ever notices. Don't add a flag-parser bound — adding "duration must be ≤ N" arbitrary numbers is more confusing than the actual behaviour. Left as documentation-only.

### 8. INFO — Signalfd lifecycle

`anuenue_animate` opens a signalfd via `_open_exit_signalfd` (line 312) and never explicitly closes it. The process exits via `syscall(SYS_EXIT)` from `main.cyr` shortly after `anuenue_animate` returns; the kernel reaps all fds at process exit.

Adding an explicit `close(sigfd)` would be cleaner but yields no observable difference. **No action required.**

### 9. INFO — Signal mask not restored after animation

`_open_exit_signalfd` calls `sys_sigprocmask(0, &mask, 0)` (SIG_BLOCK, no save of previous mask). Blocked signals stay blocked for the remainder of the process. Since anuenue exits to the shell immediately after animation completes, the mask never affects any subsequent code.

If anuenue ever grew a longer-lived caller that didn't exit immediately (none planned), the mask leak would matter. Documented here for future readers.

**No action required.**

### 10. INFO — Color override parser

`_color_override_from_str` (`src/color.cyr:54`) handles:

- `s == 0` (null) → AUTO
- `strlen(s) == 0` (empty) → AUTO
- known aliases (`auto`/`24bit`/`truecolor`/`256`/`16`/`none`/`off`/`never`) → respective mode
- everything else → AUTO

`streq` and `strlen` from Cyrius stdlib are bounds-safe. No buffer overflow even if the user passes a 4 KB nonsense value as `--color=…`.

**No action required.**
